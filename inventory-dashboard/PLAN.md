## 2. 월재고현황 실제 열 위치

- 1행: 날짜 기준 표시
- 2행: 재고 그룹명
- 3행: 단위 그룹명
- 4행: 합계 수식
- 5행: 현재고 그룹
- 6행: 본문 헤더
- 7행~105행: 실제 SKU 99개

현재 본문 구조:

| 열 | 의미 |
| --- | --- |
| A | 번호 |
| B | 코드 |
| C | 상품명 |
| D:F | 재고합계 낱개/박스/팔렛트 |
| G:I | 하은재고 낱개/박스/팔렛트 |
| J:L | 논산재고 낱개/박스/팔렛트 |
| M:P | 현재 빈 영역 |
| Q:T | 변환 기준표: 코드, 낱개/박스, 박스/파렛트, 낱개/파렛트 |
| U:Z | 현재 빈 영역 |

기존 수식:

- 재고합계 낱개: `=G행+J행`
- 하은재고 낱개: `=SUMIF('하은물류'!A:A,B행,'하은물류'!D:D)`
- 논산재고 낱개: `=SUMIF('논산합본'!$A:$A,B행,'논산합본'!$B:$B)`
- 박스/팔렛트 환산: 우측 기준표의 낱개/박스, 낱개/파렛트 참조

## 4. 업데이트 설계

기존 `월재고현황`의 D:L 수식은 직접 덮어쓰지 않는다.

- 이카운트 N004(하은물류)는 `하은물류` 탭의 A/C/D를 코드 기준으로 갱신한다.
- 이카운트 N001(논산)은 `논산합본` 탭의 A/B/C를 코드 기준으로 갱신한다.
- 시트에 있는 코드만 갱신한다.
- 외부 시스템에서 조회되지 않은 기존 코드는 0으로 덮지 않고 기존 값을 유지한다.
- 미조회 상태는 `월재고현황`의 오류메시지 열에 표시한다.

네이버재고 추가 방식:

- 현재 M:P가 비어 있고 Q:T가 기준표이므로, 최초 실행 시 Q 앞에 1개 열을 삽입한다.
- 그 결과 M:Q를 새 영역으로 사용한다.
- 기존 기준표는 R:U로 밀리며, Google Sheets가 기존 수식 참조를 자동 보정한다.
- 새 열 구조:
  - M: 네이버재고_낱개
  - N: 네이버재고_박스
  - O: 네이버재고_팔렛트
  - P: 최종수집일시
  - Q: 오류메시지

## 5. 데이터 수집 방식 결정

우선순위는 요구사항대로 유지한다.

1. 공식 API
2. CSV/Excel 다운로드 후 파싱
3. Playwright 기반 브라우저 로그인 세션 저장

현재 구현의 기본 운영 경로:

- 이카운트: `data/raw/ecount` 폴더의 최신 CSV/XLSX export 파싱
- 품고/네이버: `data/raw/poomgo` 폴더의 최신 CSV/XLSX export 파싱
- Playwright: `HEADLESS=false`로 사용자가 로그인한 세션을 `data/session`에 저장하는 보조 기능 제공

API 연동은 `ecount_client.py`, `poomgo_client.py`의 수집기 경계면을 유지했으므로, 실제 API 키와 엔드포인트가 확보되면 CSV 파서를 대체하거나 병행할 수 있다.

참고한 공식 자료:

- ECOUNT Open API: https://www.ecount.com/us/ecount/product/erp_open-api
- 품고 개발자 문서: https://developers.poomgo.com/

## 7. 운영 안전 원칙

- `.env`, 인증 파일, 세션 파일은 Git에 포함하지 않는다.
- 백업 정책 기본값은 `daily`다.
- Google Sheets 업데이트 전 `backup_YYYYMMDD_HHMMSS` 시트를 만든다.
- Google Sheets 업데이트 실패 시 로그에 남기고 실행을 종료한다.
- 개별 수집 실패는 전체 실행을 중단하지 않고 가능한 다른 소스를 계속 처리한다.
- Dashboard는 생성형 결과물이므로 매 실행 시 재생성한다.
